# C × a_∞ Analysis Figures

## Data provenance

- **Model**: Qwen/Qwen2.5-3B (base), float16
- **Format**: completion mode (see `format_conditioning_context` in `src/icl_diversity/core.py`)
- **Permutations**: 50
- **Batch size**: 16, cuda:0
- **Run tag**: `qwen25_completion_v2`
- **Compute script**: `scripts/compute_icl_metrics_for_tevet.py`
- **Compute log**: `compute_v2.log` (13,929 samples, 0 skips, 0 errors)
- **Raw results**: `results/tevet/qwen25_completion_v2/` (18 CSVs + sidecar JSONs)

## Analysis script

All figures and `summary_table.txt` generated by:
```
uv run python scripts/analyze_c_ainf.py --run-tag qwen25_completion_v2
```

Source: `scripts/analyze_c_ainf.py`. Uses `scripts/fit_ak_curves.py` for exponential fitting.

## Known data issues

1. **ConTest duplicate IDs**: Every sample_id in ConTest appears twice (once per label) with different response sets. The v2 compute used sample_id as sidecar key, so the second row overwrote the first, causing a label imbalance. Fixed in the compute script (commit TBD) by using compound sidecar keys for duplicate IDs. Affects ConTest results in this directory.

2. **McDiv only label=0**: The full McDiv CSVs have 2000 rows but only 1000 were computed, all label=0. The other 1000 (label=1) share sample_ids with label=0 rows — same duplicate ID issue as ConTest. All McDiv results show NaN.

3. **McDiv_nuggets label contamination**: 7–11 sample_ids per task appear with both labels. See Appendix B of the paper.

4. **story_gen confound**: Low-diversity story_gen responses are intrinsically more surprising to the base model due to dataset construction (paraphrases of specific/dramatic endings). See Appendix B.
